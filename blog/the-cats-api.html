Consumiendo APIs con JavaScript: The Cat API

15 de Diciembre, 2025 12 min de lectura JavaScript API Fetch Async/Await

Una de las habilidades más importantes en el desarrollo web moderno es saber consumir APIs externas. En este artículo aprenderemos a usar fetch y async/await para obtener imágenes aleatorias de gatos desde The Cat API y mostrarlas dinámicamente en nuestra página.

Objetivo del Proyecto

Este proyecto fue desarrollado en grupo con el objetivo de utilizar The Cat API para mostrar imágenes de gatos utilizando un diseño de Bootstrap. Al presionar los botones de interacción, se debe mostrar una nueva imagen obtenida desde la API.

Decisiones de Diseño
  • Utilizamos Cards de Bootstrap para simular una tarjeta estilo Instagram
  • Implementamos 3 botones con estilo icon-link: favorito, like y dislike
  • Al pulsar like o dislike se obtiene una nueva imagen desde la API

¿Qué es una API?

Una API (Application Programming Interface) es un conjunto de reglas que permite que diferentes aplicaciones se comuniquen entre sí. En el contexto web, las APIs nos permiten obtener datos de servidores externos sin necesidad de recargar la página.

Flujo de una petición API
1
Tu aplicación envía una petición HTTP al servidor
2
El servidor procesa la petición
3
El servidor envía una respuesta (generalmente en formato JSON)
4
Tu aplicación procesa los datos y actualiza la interfaz

The Cat API

The Cat API es una API pública y gratuita que proporciona imágenes aleatorias de gatos. Es perfecta para aprender porque:

  • No requiere autenticación para uso básico
  • Devuelve respuestas simples en formato JSON
  • Es rápida y confiable

Endpoint que usaremos

https://api.thecatapi.com/v1/images/search

Este endpoint devuelve un array con un objeto que contiene la URL de una imagen aleatoria:

[
  {
    "id": "II9dOZmrw",
    "url": "https://cdn2.thecatapi.com/images/II9dOZmrw.jpg",
    "width": 1080,
    "height": 1080
  }
]

Estructura HTML

Nuestra interfaz es una tarjeta de Bootstrap que muestra la imagen del gato y botones de interacción:

<div class="container py-2 px-4">

    <img src="..." class="card-img-top" alt="" id="imagen">

  </div>

Elemento Propósito

JavaScript: Fetch y Async/Await

Aquí está el código JavaScript completo que hace la magia:

async function obtenerDatos() {
  const data = await fetch('https://api.thecatapi.com/v1/images/search')
  const res = await data.json()

  const imagen = document.getElementById('imagen')
  imagen.src = res[0].url
}

Aunque son solo 6 líneas, hay mucho que entender. Vamos a desglosarlo:

1. Funciones Asíncronas (async)

async function obtenerDatos() {
  // ...
}

La palabra clave async antes de function indica que esta función:

  • Contiene operaciones asíncronas (que toman tiempo)
  • Siempre devuelve una Promesa
  • Puede usar await en su interior
¿Por qué asíncrono? Las peticiones a APIs toman tiempo (milisegundos o segundos). Si JavaScript esperara de forma síncrona, la página se congelaría hasta recibir respuesta.

2. Fetch: Haciendo la petición

const data = await fetch('https://api.thecatapi.com/v1/images/search')

fetch() es una función nativa de JavaScript que:

  • Realiza peticiones HTTP
  • Devuelve una Promesa
  • Por defecto hace peticiones GET

await pausa la ejecución hasta que la Promesa se resuelva, y devuelve el resultado (un objeto Response).

3. Parseando la respuesta JSON

const res = await data.json()

El objeto Response contiene la respuesta cruda. El método .json():

  • Extrae el cuerpo de la respuesta
  • Lo parsea como JSON
  • Devuelve una Promesa (por eso usamos await de nuevo)

4. Actualizando el DOM

const imagen = document.getElementById('imagen')
imagen.src = res[0].url

Finalmente:

  • Obtenemos la referencia al elemento <img>
  • Accedemos al primer elemento del array (res[0])
  • Extraemos la propiedad url
  • La asignamos al atributo src de la imagen

Entendiendo Promesas

Las Promesas son objetos que representan el resultado eventual de una operación asíncrona. Tienen tres estados:

Estado Descripción
pending La operación está en progreso
fulfilled La operación se completó exitosamente
rejected La operación falló

Sintaxis alternativa con .then()

Antes de async/await, se usaba .then():

function obtenerDatos() {
  fetch('https://api.thecatapi.com/v1/images/search')
    .then(data => data.json())
    .then(res => {
      const imagen = document.getElementById('imagen')
      imagen.src = res[0].url
    })
}

async/await es "azúcar sintáctica" que hace el código más legible, pero internamente funcionan igual.

Manejo de Errores

En producción, siempre debemos manejar posibles errores:

async function obtenerDatos() {
  try {
    const data = await fetch('https://api.thecatapi.com/v1/images/search')

    if (!data.ok) {
      throw new Error('Error en la petición')
    }

    const res = await data.json()

    const imagen = document.getElementById('imagen')
    imagen.src = res[0].url
  } catch (error) {
    console.error('Error:', error)
    alert('No se pudo cargar la imagen')
  }
}
Importante: fetch() solo rechaza la promesa si hay un error de red. Los errores HTTP (404, 500, etc.) no lanzan excepciones, por eso verificamos data.ok.

Opciones de Fetch

Aunque en nuestro ejemplo usamos fetch de forma básica, acepta un segundo parámetro con opciones:

fetch('https://api.example.com/data', {
  method: 'POST',           // GET, POST, PUT, DELETE
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer token123'
  },
  body: JSON.stringify({
    nombre: 'Ejemplo'
  })
})
