Tienda en Línea con Filtrado Dinámico usando JavaScript

5 de Enero, 2026 12 min de lectura JavaScript Arrays Map Filter DOM

En esta actividad de clase desarrollamos una tienda en línea completamente funcional utilizando JavaScript vanilla para aplicar los métodos de arrays más importantes: map(), filter() y forEach(). El objetivo fue crear una interfaz interactiva que permita buscar y filtrar productos de manera dinámica.

¿Qué Construimos?

Creamos una aplicación de tienda que incluye las siguientes funcionalidades:

  • Catálogo de productos: Una colección de 13 productos con imágenes, nombres, precios y descuentos
  • Búsqueda dinámica: Un buscador que filtra productos por nombre en tiempo real
  • Filtros predefinidos: Secciones especiales para productos con descuento del 12% y productos con precio mayor a $500
  • Interfaz moderna: Diseño responsivo con efectos hover y tarjetas de producto atractivas

Estructura de los Datos

Cada producto en nuestro catálogo se representa como un objeto con las siguientes propiedades: 

const products = [
  {
    imgUrl: 'https://picsum.photos/300/200?random=2',
    name: 'Laptop HP Pavilion',
    price: 800,
    discount: 0.12
  },
  {
    imgUrl: 'https://picsum.photos/300/200?random=3',
    name: 'Smartphone Samsung Galaxy',
    price: 600,
    discount: 0.05
  },
  // ... más productos
]
Nota: Los descuentos se almacenan como decimales (0.12 = 12%, 0.05 = 5%) para facilitar los cálculos matemáticos.

Métodos de Arrays Aplicados

1. Map() - Renderizado de Productos

Utilizamos map() para transformar nuestro array de productos en elementos HTML. Este método es perfecto para crear una nueva versión de nuestros datos (en este caso, strings de HTML): 

function renderProducts(products, container = null) {
  const productsContainer = container ?? document.getElementById('products')

  const rows = products.map(product => `
    <div class="product-card">
      <div class="discount" data-value="${product.discount}">
        ${Math.round(product.discount * 100)}%
      </div>
      <img src="${product.imgUrl}" alt="${product.name}">
      <div class="product-content">
        <div class="name">${product.name}</div>
        <div class="price">${product.price}$</div> 
      </div>
    </div>
  `)

  productsContainer.innerHTML = rows.join('')
}
¿Por qué map()? Porque necesitamos transformar cada producto en HTML sin modificar el array original. Map crea un nuevo array con los resultados de la transformación.

2. Filter() - Búsqueda y Filtrado

El método filter() es esencial para nuestra funcionalidad de búsqueda. Nos permite crear un nuevo array con solo los elementos que cumplan ciertos criterios: 

function buscarProducto(criterio, valor) {
  return products.filter(producto => {
    const valorPropiedad = producto[criterio]

    // Si la propiedad es un texto, hacemos búsqueda parcial
    if (typeof valorPropiedad === 'string') {
      return valorPropiedad.toLowerCase().includes(valor.toLowerCase())
    }

    // Si es un número, buscamos coincidencia exacta
    return valorPropiedad === valor
  })
}

Esta función flexible permite buscar por cualquier propiedad del producto:

  • Búsqueda por nombre: buscarProducto('name', 'laptop')
  • Búsqueda por precio exacto: buscarProducto('price', 800)

3. Filter() - Filtros Predefinidos

También aplicamos filter() para crear secciones especiales con productos específicos: 

// Productos con descuento del 12%
const productsWithDiscount = products.filter(product => 
  product.discount === 0.12
)
renderProducts(productsWithDiscount, document.getElementById('discount-12'))

// Productos con precio mayor a $500
const productsOver500 = products.filter(product => 
  product.price > 500
)
renderProducts(productsOver500, document.getElementById('discount-50'))

Manejo de Eventos con el DOM

Para hacer la búsqueda interactiva, implementamos event listeners en el formulario: 

const searchForm = document.getElementById('search-form')

// Evento de submit para buscar
searchForm.addEventListener('submit', (e) => {
  e.preventDefault()
  const searchInput = document.getElementById('search-input')

  if (!searchInput.value.trim().length) {
    return
  }

  const filteredProducts = buscarProducto('name', searchInput.value)
  renderProducts(filteredProducts)
})

// Evento de reset para limpiar búsqueda
searchForm.addEventListener('reset', () => {
  renderProducts(products)
})
Validación: Usamos trim() para evitar búsquedas con espacios vacíos y preventDefault() para evitar que el formulario recargue la página.

Características del Diseño

CSS Moderno

El proyecto utiliza características modernas de CSS para crear una interfaz atractiva:

  • CSS Grid: Layout responsivo con grid-template-columns: repeat(auto-fill, minmax(220px, 1fr))
  • Backdrop Filter: Header con efecto de vidrio esmerilado (backdrop-filter: blur(16px))
  • Sticky Header: Navegación que permanece visible al hacer scroll
  • Hover Effects: Tarjetas con animaciones y efectos de escala
  • Media Queries: Diseño adaptable para dispositivos móviles

Detalles de UX

  • Badges de descuento: Se ocultan automáticamente cuando el descuento es 0 usando [data-value="0"]
  • Validación de búsqueda: No se ejecuta la búsqueda si el input está vacío
  • Botón de limpiar: Restaura el catálogo completo con un solo clic
  • Búsqueda case-insensitive: No importa si escribes en mayúsculas o minúsculas

Conceptos Clave Aplicados

Map vs Filter vs ForEach
  • map(): Transforma cada elemento y devuelve un nuevo array
  • filter(): Selecciona elementos que cumplen una condición y devuelve un nuevo array
  • forEach(): Itera sobre elementos pero no devuelve nada (usado para efectos secundarios)

Aprendizajes y Buenas Prácticas

  1. Separación de responsabilidades: La función renderProducts() se encarga solo de renderizar, no de filtrar ni buscar.
  2. Reutilización de código: La misma función de renderizado sirve para todos los contenedores, usando el parámetro opcional container.
  3. Búsqueda flexible: La función buscarProducto() puede buscar por cualquier propiedad, haciéndola extensible para futuras necesidades.
  4. Inmutabilidad: Usamos map() y filter() que no modifican el array original, evitando efectos secundarios no deseados.
  5. Template literals: Usamos backticks para crear HTML de manera más legible y mantenible.

Conclusiones

Este proyecto nos permitió aplicar de manera práctica los métodos de arrays más importantes de JavaScript en un contexto real. Aprendimos que:

  • map() es ideal para transformar datos en visualizaciones
  • filter() es perfecto para implementar búsquedas y filtros
  • La combinación de estos métodos permite crear interfaces interactivas y dinámicas

  • CSS moderno puede crear interfaces atractivas sin necesidad de frameworks
