Implementación de Git y GitHub: Guía Completa

12 de Diciembre, 2025 15 min de lectura Git GitHub SSH

Git es una herramienta fundamental en el flujo de trabajo de los desarrolladores. En este laboratorio aprenderemos a instalar y configurar Git, crear una cuenta en GitHub, generar claves SSH y publicar nuestro primer repositorio en la nube.

Objetivo

Configurar Git y GitHub para poder versionar y compartir nuestro código de manera profesional.

¿Qué es Git? Git es un sistema de control de versiones distribuido que permite rastrear cambios en el código fuente durante el desarrollo de software.

Configuración de Git

Instalación

Descargamos el instalador desde la página oficial de Git:

https://git-scm.com/download/windows

Al completar la descarga:

  1. Ejecutamos el instalador
  2. Utilizamos todas las configuraciones por defecto
  3. Al finalizar, marcamos la casilla Launch Git Bash
  4. Presionamos Finish

Para verificar la instalación, ejecutamos:

git --version
# git version 2.52.0.windows.1

Configuración Inicial

Configuramos el nombre y correo que se usarán para las confirmaciones (commits):

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
Importante: El correo debe ser el mismo que usarás en tu cuenta de GitHub para que los commits se asocien correctamente a tu perfil.

Práctica: Tu Primer Repositorio Local

Paso 1: Crear la carpeta del proyecto

Creamos una carpeta llamada practica_git, hacemos clic derecho y seleccionamos Open Git Bash Here.

Paso 2: Inicializar el repositorio

git init
# Initialized empty Git repository in C:/Users/.../practica_git/.git/

Este comando crea una carpeta oculta .git que contiene toda la información del repositorio.

Paso 3: Crear un archivo

Abrimos VS Code con el comando code . y creamos un archivo index.html:

<!DOCTYPE html>
<html>
<body>
  <h1>Hola Mundo!</h1>
</body>
</html>

Paso 4: Ver el estado del repositorio

git status
# On branch master
# Untracked files:
#   index.html

Git nos indica que index.html existe pero no está siendo rastreado.

Paso 5: Agregar archivos al staging

git add index.html
# O para agregar todos los archivos:
git add .

Ahora el archivo está en el staging area (área de preparación), listo para ser confirmado.

Paso 6: Confirmar los cambios (commit)

git commit -m "Este es mi primer commit"
# [master (root-commit) a4e73c9] Este es mi primer commit
#  1 file changed, 1 insertion(+)
#  create mode 100644 index.html
Flujo de trabajo en Git
Working Directory
Staging Area
git add
Repository
git commit

Paso 7: Verificar el estado limpio

git status
# On branch master
# nothing to commit, working tree clean

Configuración de GitHub

GitHub es una plataforma basada en la nube donde podemos almacenar y compartir nuestros repositorios de Git.

Creación de cuenta

Para crear una cuenta, nos dirigimos a:

https://github.com/signup
Recomendación: Usa tu cuenta de correo personal, nunca una cuenta institucional o laboral (a menos que sea estrictamente necesario). Este correo debe coincidir con el configurado en Git.

Creación y configuración de clave SSH

Para subir repositorios a GitHub de forma segura, necesitamos configurar una clave SSH.

Generar la clave SSH

En Git Bash ejecutamos:

ssh-keygen -t rsa -b 4096 -C "tu@email.com"

Explicación de los parámetros:

Parámetro Descripción
-t rsa Tipo de algoritmo (RSA)
-b 4096 Cantidad de bits (mayor seguridad)
-C "email" Comentario para identificar la clave

Al ejecutar el comando, presiona Enter para aceptar la ubicación por defecto y opcionalmente agrega una contraseña.

Ver la clave pública

cat ~/.ssh/id_rsa.pub
# ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAACAQ... tu@email.com

Copiamos todo el contenido de la clave (desde ssh-rsa hasta el correo).

Agregar la clave a GitHub

  1. Ve a Settings en tu perfil de GitHub
  2. Selecciona SSH and GPG keys
  3. Clic en New SSH key
  4. Agrega un título descriptivo (ej: "Mi laptop")
  5. Pega el contenido de la clave pública
  6. Clic en Add SSH key

Creación de un repositorio en GitHub

  1. Ve a tu perfil y selecciona la pestaña Repositories
  2. Clic en el botón New
  3. Completa el formulario:
    • Repository name: nombre sin espacios ni caracteres especiales
    • Description: descripción opcional
    • Visibility: Public o Private
  4. Clic en Create repository
Nota: Para este ejercicio, no agregamos README, .gitignore ni licencia. Los agregaremos desde nuestro repositorio local.

Subir archivos al repositorio

Después de crear el repositorio en GitHub, nos muestra los comandos a ejecutar:

# Cambiar el nombre de la rama principal a 'main'
git branch -M main

# Agregar el repositorio remoto
git remote add origin git@github.com:usuario/repositorio.git

# Subir los cambios
git push -u origin main
Comando Descripción
git branch -M main Renombra la rama actual a "main"
git remote add origin URL Agrega un repositorio remoto llamado "origin"
git push -u origin main Sube los commits al repositorio remoto

Si todo está correcto, al recargar la página del repositorio en GitHub veremos nuestros archivos publicados.

Resumen de Comandos

Comando Descripción
git init Inicializa un repositorio vacío
git status Muestra el estado de los archivos
git add archivo Agrega archivos al staging area
git commit -m "mensaje" Confirma los cambios con un mensaje
git log Muestra el historial de commits
git branch Lista las ramas del repositorio
git remote -v Lista los repositorios remotos
git push Sube los commits al repositorio remoto
git pull Descarga y fusiona cambios del remoto
git clone URL Clona un repositorio existente

Conclusiones

Git es una herramienta básica en el flujo de trabajo de los desarrolladores. En este laboratorio pudimos aplicar los conocimientos básicos sobre Git y GitHub:

  • Git: Inicializar repositorios, agregar archivos y confirmar cambios
  • GitHub: Crear cuenta, registrar clave SSH, crear repositorio y publicar código
  • Flujo de trabajo: Working Directory → Staging Area → Repository → Remote
Siguiente paso: Aprender sobre ramas (branches), fusiones (merge), resolución de conflictos y flujos de trabajo colaborativos como Git Flow.
