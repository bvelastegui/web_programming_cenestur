Creando tu Primer Formulario Web con HTML5 y Bootstrap 5

13 de Diciembre, 2025 8 min de lectura HTML CSS Bootstrap

Cuando empezamos en el mundo del desarrollo web, uno de los primeros retos que enfrentamos es crear formularios funcionales y visualmente atractivos. En este artículo, analizaremos paso a paso cómo construir un formulario de operaciones matemáticas utilizando HTML5 y Bootstrap 5.

La Base de Todo: Estructura HTML5

Todo documento HTML5 comienza con una estructura fundamental que define cómo el navegador interpretará nuestro contenido:

<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Formulario HTML | Programación Web</title>
  </head>
  <body>
    <!-- Contenido aquí -->
  </body>
</html>

Los meta tags son fundamentales:

  • charset="utf-8" nos permite usar caracteres especiales como tildes y la ñ
  • viewport asegura que nuestra página se vea correctamente en dispositivos móviles

Integrando Bootstrap 5: Estilos sin Complicaciones

En lugar de escribir cientos de líneas de CSS, Bootstrap nos ofrece estilos predefinidos. Lo integramos mediante un CDN:

<link
  href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css"
  rel="stylesheet"
  integrity="sha384-sRIl4kxILFvY47J16cr9ZwB07vP4J8+LH7qKQnuqkuIAvNWLzeN8tE5YBujZqJLB"
  crossorigin="anonymous"
/>
Seguridad: El atributo integrity es una medida de seguridad llamada SRI (Subresource Integrity) que verifica que el archivo no haya sido modificado maliciosamente.

Etiquetas Semánticas: Código con Significado

HTML5 introdujo etiquetas que describen el propósito de cada sección:

<body class="container">
  <header>
    <!-- Encabezado con logo y navegación -->
  </header>

  <nav>
    <!-- Menú de navegación -->
  </nav>

  <main>
    <!-- Contenido principal -->
  </main>
</body>

Estas etiquetas no solo organizan mejor nuestro código, sino que mejoran la accesibilidad para lectores de pantalla y el SEO de nuestra página.

El Sistema de Clases de Bootstrap

Bootstrap funciona mediante clases CSS predefinidas. Veamos las más utilizadas en nuestro formulario:

Contenedores y Espaciado

Clase Función
container Centra el contenido con márgenes automáticos
py-3 Padding vertical (arriba y abajo)
mb-3 Margen inferior
px-1 Padding horizontal

Flexbox Simplificado

<div class="d-flex justify-content-between align-items-center">

Con solo tres clases activamos Flexbox, distribuimos los elementos en los extremos y los alineamos verticalmente al centro.

Componente Card

Bootstrap nos ofrece el componente Card para agrupar contenido de forma elegante:

<form class="card">
  <div class="card-header">Título</div>
  <div class="card-body">
    <!-- Campos del formulario -->
  </div>
  <div class="card-footer">
    <button class="btn btn-primary">Enviar</button>
  </div>
</form>

Construyendo el Formulario

Un formulario accesible requiere vincular cada <label> con su <input> correspondiente:

<div class="mb-3">
  <label class="form-label" for="numero1">Número 1</label>
  <input
    class="form-control"
    type="text"
    id="numero1"
    name="numero1"
  />
</div>

El atributo for del label debe coincidir con el id del input. Esto permite que al hacer clic en el texto "Número 1", el cursor se posicione automáticamente en el campo correspondiente.

Navegación con Pestañas

Para navegar entre diferentes páginas, utilizamos el componente Nav Tabs:

<nav class="nav nav-tabs">
  <a href="/FormularioHTML/index.html" class="nav-item nav-link active">
    Formulario HTML
  </a>
  <a href="/FormularioDOM/index.html" class="nav-item nav-link">
    Formulario DOM
  </a>
</nav>

La clase active indica visualmente en qué página nos encontramos actualmente.

Iconos SVG: Gráficos Escalables

En lugar de usar imágenes tradicionales para los iconos, el código utiliza SVG inline:

<svg xmlns="http://www.w3.org/2000/svg" width="20" height="20"
     fill="currentColor" viewBox="0 0 16 16">
  <path d="M8.354 1.146a.5.5 0 0 0-.708 0l-6 6A.5.5..." />
</svg>

Los SVG son vectoriales, lo que significa que se ven nítidos en cualquier tamaño y son extremadamente livianos.

Oportunidades de Mejora

Todo código puede evolucionar. Algunas mejoras para este formulario serían:

  1. Usar type="number" en lugar de text para validar automáticamente que se ingresen números
  2. Agregar el atributo required para campos obligatorios
  3. Definir un action en el formulario para indicar a dónde se envían los datos 
<input
  class="form-control"
  type="number"
  id="numero1"
  name="numero1"
  required
/>

Conclusión

Con HTML5 y Bootstrap 5 podemos crear formularios profesionales en poco tiempo. Los conceptos clave que aplicamos fueron:

  • Estructura semántica de HTML5
  • Integración de frameworks CSS vía CDN
  • Sistema de clases utilitarias de Bootstrap
  • Componentes predefinidos (Cards, Nav Tabs, Forms)
  • Accesibilidad mediante labels vinculados
Siguiente paso: Agregar JavaScript para procesar los datos del formulario y realizar las operaciones matemáticas.
